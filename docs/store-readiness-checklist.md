# Microsoft Store Readiness Checklist

DimToOff should target Microsoft Store MSI/EXE distribution first. MSIX can be revisited later if the app is reworked into a packaged desktop app.

## Current Recommendation

Use Inno Setup for the first Store-oriented installer.

Reasons:

- The app already publishes as a folder with `DimToOff.exe`, `DimToOff.Settings.exe`, XAML artifacts, and native Windows App SDK assets.
- Inno Setup can install that folder layout without reshaping the app.
- Per-user install is straightforward with `PrivilegesRequired=lowest` and `{localappdata}\Programs\DimToOff`.
- Start Menu shortcuts and uninstall support are simple.
- It is easier to iterate than WiX/MSI while the product and UI are still changing.

WiX/MSI may be worth revisiting later if enterprise deployment, MSI transforms, or stricter Windows Installer integration become important.

## Before Partner Center Submission

- [ ] Create or verify the Microsoft Partner Center developer account.
- [ ] Reserve the app name `DimToOff`.
- [ ] Decide whether the Store submission uses the standalone installer or the smaller dependency-based installer.
- [ ] Publish `PRIVACY.md` on an HTTPS URL.
- [ ] Confirm the support URL, likely GitHub Issues.
- [ ] Prepare Store screenshots.
- [ ] Prepare app icon assets for Store listing.
- [ ] Prepare age rating answers.
- [ ] Prepare Store description, keywords, and certification notes from `docs/store-listing.md`.
- [ ] Confirm whether the app name, publisher name, screenshots, and descriptions are consistent.

## Installer Requirements and Checks

- [ ] Build the standalone release package with `tools\Publish-Release.ps1`.
- [ ] Build the Inno Setup installer with `tools\Build-Installer.ps1`.
- [ ] Confirm the installer installs to a per-user location.
- [ ] Confirm the installer does not require administrator privileges.
- [ ] Confirm the installer places `DimToOff.exe` and `DimToOff.Settings.exe` in the same directory.
- [ ] Confirm the installed app can open Settings from the tray icon.
- [ ] Confirm the installed app can open the tray quick panel.
- [ ] Confirm the Start Menu shortcut launches `DimToOff.exe`.
- [ ] Confirm uninstall removes installed app files.
- [ ] Confirm uninstall removes the `HKCU\Software\Microsoft\Windows\CurrentVersion\Run\DimToOff` value if present.
- [ ] Decide whether user settings/logs should remain after uninstall. Current policy keeps them unless manually deleted.
- [ ] Document manual data removal in the privacy policy.
- [ ] Confirm silent install works:

```powershell
.\DimToOff-<version>-setup.exe /VERYSILENT /SUPPRESSMSGBOXES /NORESTART
```

- [ ] Confirm silent uninstall works from the uninstall command generated by Inno Setup.
- [ ] Host the Store installer at a stable HTTPS URL that does not change for that version.
- [ ] Keep the installer URL version-specific; do not replace an existing versioned URL with different bits.
- [ ] Sign the installer with a code-signing certificate that chains to a trusted root.
- [ ] If the installer downloads anything at install time, remove that behavior. The installer should be self-contained for Store review.

## App Behavior Disclosures

- [ ] Explain that the default mode is a blackout overlay that keeps Windows awake and unlocked.
- [ ] Explain that OLED and mini-LED displays benefit most.
- [ ] Explain that typical LCD backlights may remain on.
- [ ] Explain that `MonitorPower` mode may lock, sleep, or affect multiple displays on some machines.
- [ ] Explain WMI brightness monitoring.
- [ ] Explain low-level input hooks.
- [ ] State clearly that key contents are not stored.
- [ ] State clearly that mouse coordinates and input history are not stored.
- [ ] State clearly that there is no telemetry or network communication.

## Local Validation

- [ ] Install on a clean Windows 10 or Windows 11 test user.
- [ ] Run without administrator privileges.
- [ ] Start app from Start Menu.
- [ ] Open Settings from tray left-click.
- [ ] Open quick panel from tray right-click.
- [ ] Enable and disable Start with Windows.
- [ ] Lower brightness to trigger Blackout mode.
- [ ] Restore with keyboard, mouse, and touchpad input.
- [ ] Verify audio/background work continues during Blackout mode.
- [ ] Exit from tray and confirm `DimToOff.exe` and `DimToOff.Settings.exe` exit.
- [ ] Uninstall and confirm installed files are removed.
- [ ] Run Windows App Certification Kit where applicable.
